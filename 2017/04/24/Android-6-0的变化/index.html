Android 6.0的变化 

上周把病历夹Android版本的compileSdkVersion由21升级到了23，也就是Android6.0。说来惭愧，Android6.0已经发布近一年，直到现在才拥抱了新变化。。。

下面简单总结一下升级后遇到的问题。
首先修改build.gradle的配置以后，编译了一下工程，报了上百个错误，在若干只羊驼从脑海里飞奔而过以后，开始一个一个的检查，大概看了一下，问题实际上就那么几个，主要是用到的地方太多。

1.移除Apache Http Client
6.0发布版移除了对Apache Http Client的支持，官方推荐使用HttpUrlConnection类来替换（targetSdkVersion>9）,因为后者使用了对用户透明的压缩和对响应的缓存，最大限度的减少了电量的消耗。如果要继续使用Apache Http Client的相关api，只需要在gradle中声明编译期依赖：

2.通知
新版本移除了Notification.setLatestEventInfo()方法，使用Notification.Builder 类来代替构造方法。

3.FloatMath类
移除了FloatMath类，在一些自定义view中有使用，直接用Math类替换即可，官方的说法是在运行JIT编译器的安卓设备上Math类的效率要远高于FloatMath类。

其他的重大变化：

1.权限机制
可以说是Android6.0最大的亮点。也呼应了Android对于用户安全的重视。Android6.0引入了全新的权限机制，用户可以在运行时直接管理某个应用程序的权限。也就是应用程序要在使用到某个权限时，事先检查是否获取到了这个权限，通过checkSelfPermission()方法,通过requestPermissions()方法来获取权限.

2.Doze和应用挂起
为空闲的设备和应用程序引入了电源节能的优化。
如果用户拔掉了设备的电源线，并且设备是静止的，当屏幕关闭一段时间以后，系统将进入Doze模式。这时系统进入休眠状态，这种模式下设备会周期性的恢复某些normal的操作，以便应用程序的同步，同时系统可以执行一些耗时的操作。
当用户一段时间内没有使用某个应用程序，系统将把这个应用视为挂起状态，将禁止应用访问网络，并停止该应用的操作。

3.WIFI和网络变更
应用程序只能修改或者删除由自己创建的WifiConfiguration对象，由用户或者其他应用程序创建的WifiConfiguration对象你是没有权利修改删除的。在之前的版本中，如果应用程序使用 enableNetwork() ，并设置 disableAllOthers=true来强制设备连接到指定的 Wi-Fi 网络，设备将和其他网络断开。这个版本中，这些设备将不再和其他网络断开。如果应用程序的 targetSdkVersion 是 20 或者更低，则会连接被选中的 Wi-Fi 网络。如果应用程序的 targetSdkVersion 是 21 或者更高，使用多网络 API (如 openConnection()， bindSocket() 及新的 bindProcessToNetwork() 方法)来确保它的网络通信是发给被选中的网络。

4.摄像头服务变更
访问摄像头服务的共享资源时，优先级发生了变更。不再是“先到先得”，而是具有高优先级的进城优先获得。

5.USB连接
通过 USB 端口的设备连接默认为充电模式。想要通过 USB 连接来访问设备和它的内容，用户需要为这些交互提供明确的授权。如果你的应用程序支持用户通过 USB 端口与设备交互，需要确保这些交互被明确启用。
